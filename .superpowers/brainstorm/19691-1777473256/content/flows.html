User Flows

Auth, booking, and cancellation

Sign In / Register
Visit /yoga or click "My Account"
→
/login page (email + password)
→
First time? Enter name + phone too
→
Redirected back to previous page

Single page handles both login and register — toggled by a tab. Supabase sends a confirmation email on register.

Booking a Session
Click "Book" on /yoga
→
Not logged in? → /login then back
→
Confirm modal: session details
→
Booked! Button → "You're in"

If session is full, button shows "Full" and is disabled. If user already booked, button shows "You're in".

Cancelling a Booking
Go to /profile → My Bookings
→
Click "Cancel" on upcoming booking
→
Confirm cancel modal
→
Spot freed, booking removed

Cancelling sets booking status to 'cancelled' (not deleted) so the admin can see historical data. The freed spot becomes available immediately.

Admin: Creating a Session
/profile → Admin → Sessions
→
"+ Add Session" or "+ Add Recurring"
→
Fill form (title, date/recurrence, time, duration, capacity, instructor)
→
Session(s) appear on /yoga

Recurring sessions generate instances 8 weeks ahead. Admin can edit any individual instance without affecting the recurring template.
